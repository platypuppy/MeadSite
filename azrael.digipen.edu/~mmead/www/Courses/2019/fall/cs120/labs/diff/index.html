Information

Experimenting with the command-line diff and visual diff tools:

  1. Below are several text files for you to use. Each pair has a "correct" output and a student's output. Look at each pair of files to see if you can spot the differences with your eyes. You'll find that it can be very difficult to spot the differences.

    Correct output #1    Student output #1

    Correct output #2    Student output #2

    Correct output #3    Student output #3

    All output in one zip file: alloutput.zip

    Then, compare them using the diff program, which is part of the Cygwin utilities on Windows, but is built-in to Linux and Mac. For example, to compare the first set of files, you would type this at the command prompt: 

    diff output.master1.txt student-output1.txt
    This will display the differences to the screen. If you want to capture the differences in a file (for later viewing), you need to use output redirection so the output will be placed into a file like this: 
    diff output.master1.txt student-output1.txt > differences.txt
    The greater-than symbol, >, is the output redirection operator used by the command window.

    This will place the output in a file called differences.txt. You can choose to name the file anything you like. This is exactly how I can tell if you did your programming assignments correctly.

    At this point, it's not important that you understand exactly how diff works. You should, however, be able to see where the lines are different. Using Notepad++ (or any text editor), modify the student output file (add lines, remove lines, change lines) and see how diff reacts.

    Note: You may need to use the --strip-trailing-cr option with diff if your end-of-line characters are different. 

    diff --strip-trailing-cr output.master1.txt student-output1.txt > differences.txt

    If you're an advanced student, you may want to learn more about diff by reading Comparing and Merging Files with GNU diff and patch, which contains detailed information about this program.

  2. After you've seen how diff works, compare the files using WinMerge. WinMerge is a graphical program for Windows that displays the files side-by-side and highlights the lines that are different. The program is installed on the computers at Digipen. You can download it from the WinMerge Website if you'd like to run it on your own computer.

    Run WinMergeU (from the Start menu, choose Run... and type winmergeu) and drag and drop the two files onto the program. (The actual name of the executable in Windows is WinMergeU.exe. The U means it supports Unicode. More and more.) Or, you can select two files in Explorer, right-click on one of them and choose WinMerge. It will be obvious where the differences are.

    If you have put WinMerge in your path (the installer will do this automatically for you), you could compare them by typing this at the command prompt: 

    winmergeu output.master1.txt student-output1.txt
    Use the "up" and "down" arrows at the top of the program to navigate between differences in the files.

    If you're working on a Mac, you can read about different tools here:

    Diff Tools on macOS and Mac equivalents of WinMerge

    If you're working under Linux, there are several similar tools available, such as meld, kdiff3, diffuse and kompare and you should be able to install them from your distribution's package manager.

    These are all free, cross-platform and should work with Windows, macOS, and Linux:

    meld
    diffmerge
    kdiff3

    Personally, I use Beyond Compare. It supports Linux, Mac, and Windows. However, it's not free.

    If you need help with any of these diff programs, please ask on the Miscellaneous Topics forum in Moodle. There may be other students that are more familiar with other operating systems that can help you.
