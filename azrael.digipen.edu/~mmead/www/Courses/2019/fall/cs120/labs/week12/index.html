CS 120 - Lab Assignment

Information

As always, please read ALL of the information before writing any code. Most of the problems that student's have are a result of writing code without reading all of the instructions and information.

In this lab, you will write functions that will manipulate singly-linked lists. Refer to the linked list notes for more information. (A lot of code is posted there for you to use.) Also, you can find more information in chapter 17 of the textbook.
  1. There are 7 functions that you need to implement for this lab. Here are the functions with a brief description for each. The approximate number of lines of code for each is listed at the end of the description.

    FunctionDescription
    void
    add_front(struct Node **list, int value);    		 Given a list, add a new node to the front of the list that contains value. (~ 3 lines of code)
    void
    print_list(const struct Node *list);    		 Given a list, print the value of each node. The format specifier for the printf function is "%3i". (~ 4 lines of code)
    void
    free_list(struct Node *list);    		 Given a list, free all of the nodes on the list. (~ 4 lines of code)
    int
    size_list(const struct Node *list);    		 Given a list, count the number of nodes in the list and return that value. (~ 5 lines of code)
    int
    sum_list(const struct Node *list);    		 Given a list, add up all of the values in the nodes and return that value. (~ 5 lines of code)
    void
    add_back(struct Node **list, int value);    		 Given a list, add a new node to the back of the list that contains value. (~ 8 lines of code)
    void
    remove_item(struct Node **list, int value);    		 Given a list, remove the node that contains value. If there are multiple nodes that contain the same value, you will only remove the first one that you encounter. Don't forget to free the memory for this node or you'll have memory leaks! (~ 16 lines of code)

    Notes:

    1. All of the functions take a pointer (or a pointer to pointer) to a Node struct as the first parameter.

    2. Three of the functions can potentially modify the front of the list. These functions take a pointer to pointer to a Node.

    3. All functions must be able to handle the situation when an empty list (NULL) is passed in.

    4. You should work on the functions in this order:

      1. add_front
      2. print_list
      3. free_list
      4. size_list
      5. sum_list
      6. add_back
      7. remove_item

      The functions above are listed in order of their difficulty to implement. Also, the first 3 functions are required for every test in the driver. Failure to properly implement these functions will cause you to fail every test.

    5. DRAW DIAGRAMS! Neither myself nor any TA will help anyone that does not have a diagram that shows the logic that you are trying to implement. (If you feel that you don't need diagrams, then you don't need any help. See how that works?)

    6. Pay close attention to your diagrams and code when you are manipulating the first node in the list. This is where most students will have problems.

    7. Get into the habit of using while loops when working with linked list. Yes, they can be manipulated with for loops, but you'll rarely see that outside of a freshman course.

    8. Be sure to properly free the memory used by the nodes. There are two functions where you will need to free nodes: free_list, and remove_item. If you forget to free any nodes, you'll have a memory leak and the code won't be accepted.
