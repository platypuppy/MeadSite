  3. Pseudocode There are a variety of ways to approach this. This is just an overview of the things that you are going to do. Some of the details have been left out, but you should be able to write C++ code from the description below. As always DRAW DIAGRAMS. You will not get any help from a TA if you are working without diagrams. 
    Setup a pointer to the first node of the left list. (left) 
Setup a pointer to the first node of the right list (right)
Setup a pointer to point to the head of the new list. (This pointer will only be set once and will initially point to NULL.)
Setup a pointer to point to the tail of the new list. (This will be updated every time you add a node to the new list and will initially point to NULL).

While both pointers (left and right) are pointing at a node
  Compare the value of the left node with the right node
  If the left node is less than or equal to the right node, 
    Add the left node to the new list
    Move the left pointer to the next node in the left list
  Else  (The left node is greater than the right node)
    Add the right node to the new list
    Move the right pointer to the next node in the right list
  End If
End While

If the right side is empty and the left side still has nodes
  Add all of the nodes from the left side to the end of the new list
If the left side is empty and the right side still has nodes
  Add all of the nodes from the right side to the end of the new list

Make sure to update any other members before returning.
    Other hints:

    • Make sure you can handle the case when either the left or right list is empty.
    • The lists are unlikely to be the same length, so don't assume they will be.
    • Make sure the rhs list is empty afterwards. If you are storing a counter in that list, make sure you set the count to 0.
