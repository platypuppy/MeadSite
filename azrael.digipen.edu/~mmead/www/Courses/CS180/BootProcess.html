The Boot Process

Starting the Computer

A computer won't do anything until it is powered-up.
  • What happens when the computer is turned on?
  • How does the computer "know" where the operating system is located?
  • What if the computer has more than one operating system installed?
    • Which OS will be started?
    • Can the user change this?
    • What if the user wants to run multiple operating systems at the same time?

The Power-On Self Test (POST)

  • The computer first runs a self-diagnostic or POSTPOST. (Not unique to computers.)
  • Happens automatically (It is hard-coded into the BIOSBIOS and always executes first.)
    • Newer computers may use Unified Extensible Firmware InterfaceUnified Extensible Firmware Interface or UEFI.
  • Discovers what kinds of devices are connected and checks if they're in working condition.
    • CPU
    • Main memory
    • Disk drives (hard, floppy, DVD, etc.)
    • etc...
  • Reads information stored in the BIOS (typically part of the motherboard) and initializes other parts.
  • Some hardware devices (e.g. video cards) may do their own POST at this time as well.
  • In the olden-days, if there were problems you would hear beep codesbeep codes. (No other way to report problems.)
  • Newer motherboards have LED digits that may provide more information.
    • Some even use "voice" output to identify the problem.
  • Warm start vs. cold start (reset)
  • If everything checks out, the next steps takes place.

Finding and loading an operating system

  • By default, the BIOS runs a program that is stored in the first sector of the first hard disk.
    • This sector contains the Master Boot RecordMaster Boot Record (MBR)
    • Some operating systems will use a GUID Partition TableGUID Partition Table (GPT) on very large (over 2 TB) disks.
  • This sector is typically only 512 bytes, too small for an operating system.
  • This program, called bootstrapping code or bootloaderbootloader, contains just enough code to "find" an operating system elsewhere.
  • Typically, the bootloader finds the first sector (Volume Boot Record) of the active partition, then loads and executes the code.
  • Computers with multiple operating systems may display a menu for the user to choose an OS to start.
  • You can alter the location of the bootloader in the BIOS (so you can boot from a DVD, USB drive, or over the network, for example.)
  • Remember that code must be in memory to be executed.

The Master Boot Record (MBR)

The MBR is 512 bytes:
  • The first 446 bytes contain the primary boot loader code, with the last 6 bytes being a disk signature (typically only 4 bytes are used). (lavender)
  • The next 64 bytes contain the partition tablepartition table. Four records, each 16 bytes long. (pink)
    • Byte 1: Active/bootable flagActive/bootable flag. 80h is active/bootable, 00h is not.
    • Bytes 2-4: The starting cylinder-head-sectorcylinder-head-sector (CHS).
    • Byte 5: Partition typePartition type (System IDs)
    • Bytes 6-8: The ending cylinder-head-sector (CHS).
    • Bytes 9-12: The starting sector using Logical Block AddressingLogical Block Addressing (LBA).
    • Bytes 13-16: The number of sectors in the partition.
  • The last 2 bytes are the Magic Number and are always 0xAA55 (little-endian). (violet)
  • You can see some minimal error messages that are stored in the code. (turquoise)
  • The MBR is not in any partition. (It comes before the first partition.)
  • The first sector of a partition is called the Volume Boot RecordVolume Boot Record.

GUID Partition Table


The Master Boot Record technique was designed many years ago and is running into some limitations. 

    
	
  • The MBR only allows 4 primary partitions or 3 primary and 1 extended partition
		(with logical drives in the extended partition). 
		Additional information.
    • 
	
  • The partition table in the MBR uses 32 bits for block addresses.
    • 
	
  • This means a disk can have at most 232 blocks. With 512-byte blocks (sectors) this tells
		us that the maximum disk size is about 2 TB. (232 * 512 = approximately 2 TB)
    • 
	
  • A sector size of 4,096 bytes will give us a maximum disk size of about 16 TB, plenty for the next
		several years (but it's still going to be a problem then).



A new technique was created to overcome these limitations:

    
	
  • In the late 1990s, Intel invented a new technique called the 
		GUID Partition TableGUID Partition Table
		 or GPT.
    •  It is part of the Extensible Firmware Interface (from Intel) that has now become the
		 Unified Extensible Firmware InterfaceUnified Extensible Firmware Interface or UEFI.
		 
	
  • The GPT header begins on the second logical block of the hard drive (for backward-compatibility with the MBR scheme)
		and includes a pointer to the partition table (called the Partition Entry Array).
    • 
		
      
			
    • The first block is referred to as the "protective MBR" so that older disk utilities don't get
				confused and think that there are no partitions on the drive.
      • 
			
    
	
  • The default minimum number of partitions possible with a GPT is 128. (Windows sets this as the maximum.)
    • 
	
  • A copy of the header and partition table are written to the beginning and end of the disk, for redundancy in
		case of disk corruption.
    • 
	
  • All 32-bit and 64-bit version of Linux support GPT (since about 2008)
    • 
	
  • All 32-bit and 64-bit version of Mac OS X (Intel-based hardware) support GPT (since about 2008)
    • 
	
  • 32-bit Windows (before Windows 8) doesn't support booting with GPT.
    • 
	
  • 64-bit Windows (Vista and later) supports booting, but requires special UEFI (modern) hardware.
