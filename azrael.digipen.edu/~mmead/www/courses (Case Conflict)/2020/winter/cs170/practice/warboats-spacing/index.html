Practice Assignment

(warboats-spacing)

Information

This practice assignment extends the previous practice assignment that allows for boats of variable sizes. You must use that as the starting point for this practice. It also supports diagonal boat placement, but you don't have to run those tests if you haven't implemented that functionality yet.

  1. This practice assignment is a modification of the WarBoatsOO (Object-oriented WarBoats) assignment. In the original WarBoats program, all of the boats were placed anywhere inside the Ocean as long as they didn't overlap with any other boats. Obviously, this makes sense.

    The modification you are going to make for this practice is that you will not allow any boat to "touch" any other boat. This means that the boats must have at least one "space" or "cell" between themselves and other boats.

    To support this feature, the constructor for the Ocean class has changed to this in the header file: 

    Ocean(int num_boats, int x_quadrants, int y_quadrants, bool force_space = false);
    The third parameter will determine if you should leave a space between the boats. You'll need to add a private boolean member variable that holds the value of force_space so that you can reference it in your PlaceBoat method. The reason for the default value is for backward-compatibility with the original class.

    There are 6 tests in the driver and you need to provide the test number you wish to run on the command line. There is also a second command line option that lets you specify whether or not to have spacing between the boats.

    Examples:

    This will run test #2 without forced spacing: 

    warboats-spacing 2 0

    This will run test #4 with forced spacing: 

    warboats-spacing 4 1
