Programming Assignment #4 FAQ

(Frequently Asked Questions)


Here are some popular Questions and Answers for the List assignment.

  1. Question:
    What should the pop_front() method do if the list is empty?

    Answer:
    Nothing. You should check to make sure it isn't empty before trying to remove the first node.

  2. Question:
    What should the front() method return if the list is empty?

    Answer:
    It should return the value in the first node. If there is no first node, then your program should crash. DO NOT check to see if the list is empty. You should simply return the value that the head pointer is pointing to. If head points to NULL (which it will when the list is empty), then you should dereference the NULL pointer causing your program to crash. One reason for this seemingly odd behavior is this method must return a value. If you detect that the list is empty, you're still going to have to return something, so just let it crash.

    THAT IS THE CORRECT BEHAVIOR WHEN THE USER DOES STUPID THINGS.
    (Moral of the story: Don't do stupid things.)

    In the future, we will learn about exception handling. That is the proper way to deal with this "exceptional" situation (if you actually want to deal with it).

  3. Question:
    What should we do if the index provided to the overloaded subscript operators is invalid?

    Answer:
    You should do nothing. That's right, just "walk off the end of the list..." trashing everything and anything along the way! That's what happens in the Real World™. When we talk about exception handling, we'll see a better way to handle this "exceptional" case.

  4. Question:
    How does the clear method work? The sample driver isn't calling it.

    Answer:
    The handout describes what the clear method should do. You should be able to figure out what the prototype of the function will be. (Note, it doesn't return anything.) Even though the provided sample driver doesn't call it, your code should. (Hint: The destructor should call it. That's code re-use.)

  5. Question:
    The programs that I compiled with g++ and clang++ give me the same value for the total number of lists created, but Microsoft's compiler gives me a different number. Am I doing something wrong?

    Answer:
    No. You may get different answers. Bonus to those students that know why. (This is an advanced topic covered in the CS 225 course, although I may have mentioned it during class.)

  6. Question:

  7. Question:
    Why do we need two subscript operators (operator[])?

    Answer:
    Because the driver won't compile with just one. You can find the answers in the required reading and in the notes.
