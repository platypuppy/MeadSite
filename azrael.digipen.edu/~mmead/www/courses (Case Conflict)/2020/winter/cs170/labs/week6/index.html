Notice that none of these methods are marked const. That's because all of these methods modify the list.

Also notice that the Insert and Delete methods are overloaded:

  • The first Insert method will insert the value at the position supplied. You've already done this in a previous lab, but it is overloaded now. See the test driver and output below for more examples.
    • Note that if position is greater than the number of items in the list (or negative), nothing is inserted. This means that you need to make sure that the insert position is valid.
  • The second Insert method takes just a value. It will insert the integer based on its value. This will allow the user to keep the list sorted. So, if I insert the values 4, 7, 1, 9, 2 in that order, the list will look like this:
    1 → 2 → 4 → 7 → 9
  • Note that this will only work if the second Insert is used to add all of the items. If you call the second Insert when the list is not sorted, you won't get the correct results. That's fine. We are NOT going to try and do a sorted insert into an unsorted list at this point. The driver is not going to mix the two Insert methods on the same list.

  • The first Delete method searches (from the beginning of the list) for the first occurrence of value and removes it from the list. If there is no occurrence of value, nothing is deleted.
  • The second Delete method takes a second List. For each item on the second list, all occurrences of that item/value are deleted from the object. See the test driver and output to see how it works.

  • The Concat method must allocate a new node for each node in List2. If you just point the last node from List1 at the first node in List2, you are going to have serious problems when the destructors for each list are called.

    Hints:

    1. Since you are not using PrintList anymore (you are overloading operator<<), you must implement this first. A partial implementation is in the .cpp file already. All of the tests in the driver rely on this function.
    2. If you create a private helper function that finds an item in the list and also returns the previous node in the list, you will be able to use that function in FindItem and also in the first Delete function. This will make your code shorter and easier to write and maintain.
    3. The second Delete function should call the first one. (That's why it requires the least amount of code.) It won't be the most efficient method, but it will make your code easier to read, write, and debug. This is the primary goal during the first year of programming: Correctness and Readability.

      This is also related to the concept of Premature optimization is the root of all evil and is a trap that many beginning programmers fall into.

    4. Remember, if you create helper functions, put them in the private section so they are hidden from the clients.
    5. You can work on the functions in any order you like.
    6. Make sure you are freeing the deleted nodes (using delete) or you'll have memory leaks.
    7. Use a memory debugger to verify that you have no memory leaks.

  2. Add a Doxygen file header comment to List.cpp. Make a bullet list of the functions in the file. Create doxygen function header comments for each function.
