CS 170 - Lab Assignment

Information

For this week's lab, you will take your previous lab implementations for a linked list and create a class out of them. There really isn't much new functionality, so this lab won't take you as much time.

Note that all of the functions will need to be changed. This is due to the fact that you are no longer passing a List to them. The functions and data (the List) are part of the class. Also, please be sure to use const where appropriate. This means using it for function parameters that won't be modified, as well as marking member functions with const if they are not modifying any of the data in the object.

  1. Create a class called List that contains all of the global functions (as member functions now) for the lists that you've been writing so far. Take a look at the test driver (driver.cpp   HTML) if you want to see how all of the functions are being called. Here is a starting point: (List.h) 
    ////////////////////////////////////////////////////////////////////////////////
#ifndef LIST_H
#define LIST_H
////////////////////////////////////////////////////////////////////////////////

//! namespace CS170
namespace CS170
{
  //! namespace ListLab
  namespace ListLab
  {
    //! Node structure for List class
    struct Node
    {
      
      int number; //!< data portion
      Node *next; //!< pointer portion 

      /*!*******************************************************************
        Conversion constructor

        \param value
          The data for the node
      **********************************************************************
      Node(int value)
      {
        number = value;
        next = 0;
      }
    };

    //! Single-linked list class for CS170 labs
    class List
    {
      public:
          // Default constructor
        List();

          // Destructor
        ~List();

        // Other public method declarations here ...
        
        // DO NOT implement any methods here.
        // They must be implemented in the .cpp file.

      private:
        Node *head_;  //!< Pointer to the front of the list
        Node* MakeNode(int value) const;
        
        // Any other private helper methods you need...
    };
  } // namespace ListLab
} // namespace CS170

#endif // LIST_H
    Implement all of the methods in a file called List.cpp. (Notice the capitalization.) You'll have to remove some methods. For example, since you now have a destructor, you need to remove the FreeList method (or make it private, if you still wish to use it).

    Here is the starting point for the implementation file: List.cpp

  2. Add a Doxygen file header comment to List.cpp and List.h. Make a bullet list of the functions in the file. Create doxygen function header comments for each function. You already have Doxygen comments for most of the functions, so it's just a matter of modifying them to match the member functions. Make sure you document all functions, even ones that were given to you.

    Notice the Doxygen comments in the header file for the data (struct, class, etc.) You will need these or Doxygen will complain. I will not always provide these comments for you and you will need to provide them yourselves. Don't wait until the end to add Doxygen comments and then find that the submission server rejects your code because you failed to do this.

    Consult the Doxygen help file if you haven't yet. (It was assigned as homework 4 weeks ago.)
