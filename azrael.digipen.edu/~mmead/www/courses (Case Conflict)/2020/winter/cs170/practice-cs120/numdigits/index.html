Practice Assignment

(numdigits.c)

Information

  1. Write a program that prints out the number of digits in an integer. Here are a few sample runs: 
    Enter an integer (1 to 2,147,483,647 without commas): 1776
The number of digits in 1776 is 4.

Enter an integer (1 to 2,147,483,647 without commas): 8
The number of digits in 8 is 1.

Enter an integer (1 to 2,147,483,647 without commas): 1357902468                  
The number of digits in 1357902468 is 10.
    The numbers in RED are typed by the user.

    The name of the file should be numdigits.c and the command to compile it will look like this: 

    gcc -Werror -Wall -Wextra -ansi -pedantic numdigits.c -o numdigits
    output-numdigits.txt This output was generated by running the program 9 times, using each of these numbers: 
    1

Notes

  • Approximate number of lines of code: 10.

  • You must use a loop (for, while, etc.) and can not use any conditionals (if, switch, etc.)

  • Hint: You will only need to use the division operator (repeatedly) on the value to get the number of digits. For example, if the number is 1234, dividing it by 10 gives you 123 (integer division). Dividing by 10 again gives 12. When the result is 0, you've divided the number X times, where X is the number of digits in the number.

  • You don't have to worry about invalid input. You can assume that the user is going to input an integer within the specified range.

  • You are only prompting the user once, printing the number of digits, and then ending.

  • The first time you redirect to a file, use a single greater-than symbol (>): 
    numdigits > myoutput.txt
    On subsequent redirects, use two greater-than symbols with no space between them (>>). 
    numdigits >> myoutput.txt
    This will append to the existing file, rather than overwrite it. Of course, you won't see any prompt because that, too, is being redirected to the file.

  • See a video showing the behavior of the program.

For students that want to practice their C-fu

  • Re-prompt the user on invalid input (using some kind of loop, preferably a do...while).
  • Allow negative numbers.
  • Allow floating point numbers. This will require a different technique and will be much easier to do if you convert the number to a string at first.
  • NEVER UPLOAD ANY CODE THAT DEVIATES FROM THE "NORMAL" CODING ABOVE.
